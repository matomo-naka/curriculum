『ITとは』
Information Technologyの略で、パソコンやスマートフォンなどの情報技術の総称。
『OSとは』
Operation System の略で、ハードウェアとソフトウェアの仲介する役割のあるもの。
『Windows』
マイクロソフト社が開発したOS
『Mac OS』
Apple社が開発したOS
『Unix』
現存するOSの中で最も古いOS
『Linux』
Unixベースで開発された無料のOS

『HTML』
Hyper Text Markup Language 文字などを表示させる言語
『CSS』
Cascading Style Sheets 『HTML』でコードした物の見た目を変える
『MySQL』
データの集まりである、データベースを管理するシステム。
『PHP』
Hypertext Preprocessorの略で、動的にWebページを生成できる。Wordpressにも使われ、汎用性が高い
『JavaScript』
サイトに動きをつけたり、マルチメディアを管理したりする便利な言語。
『jQuery』
JavaScriptのプログラムで、Web上で一番使われている。HTML、CSSを簡単に操作でき、どのブラウザでも使える。
『Apache』
Webサーバーソフトウェアの一つで、セキュリティ面でもよく、OSSで誰でも利用可能。PHPのデータベースとの連携がとりやすい。
『Bootstrap』
デザインにかける時間を大幅に短縮できる。画面のレイアウトがすぐ作れる。

『SE』
System Engineer の略、要件定義、基本設計、テストなど幅広く上流工程も担当する。
『PG』
PGからSE、SEから指示される仕様書に基づいて開発を担当する人々
『Webデザイナー』
見た目の良いデザインを作るエンジニア。Photoshopなどを使用する。

『CUI』
コマンドラインユーザーインターフェイス、コマンドでOSとやりとりする技術
『GUI』
視覚的、体感的にOSとやりとり、例えばマウス操作、タッチパネルなど

『ファイル』
テキストファイルなど、内容の書かれた物のこと。それぞれのデータが書かれている。
『ディレクトリ』
ファイルをまとめたフォルダのようなもの。

『ルートディレクトリ』
現在いるファイルや、ディレクトリの外がわ、一番上のディレクトリ
『カレントディレクトリ』
現在いるディレクトリのこと
『パス』
目的のファイルにたどり着くための経路

『絶対パス』＝ Uniform Resource Locator
絶対に変わることのないパス。ドメイン名、フォルダ、ファイル名が指定されている。URLのこと
『相対パス』
今いる場所を起点に、目的のファイルを指定した書き方。


『HTTP』
Web情でWebサーバーとクライアント間でやりとりするプロトコル。

『FTP』
サーバー、クライアント間で、負ファイルを送受信する通信するプロトコル。

『HTTPS』
HTTP通信をより安全に行うためのプロトコール

『SSL』＝ Secure Sockets Layer
重要な情報を盗まれたり改竄されないようにする仕組み。

『SSH』
Secure Shell　を用いる通信は暗号化されていて安全に通信できる仕組み

『ポート番号』
その番号で通信するサーバーに振り分けられる。

『HTTPS：TCP 443 SSH: TCP 22 FTP: TCP 20』

『TCP』
Transmission control protocol 毎度通信を確認する仕組み。

『ファイアウォール』
外部ネットワークを不正アクセスから真木ための防火壁。接続を毎回不正かどうか判断できる。

『IPアドレス』Internaet Protocol address
それぞれのインターネットに割り振られたアドレス。
『グローバルIPA』
それぞれのデバイスに割り振られたアドレス
『プライベートIPA』
プロバイダからもらうアドレスで、特に自宅のルーターのアドレスに割り振られる。

『ドメイン』
インターネットの住所。名前　...com
『DNS』Domain name system
IPアドレスとドメイン の紐付けサービス
『VPN』
Virtual Private Server 仮想サーバーを提供するサービス
『ブリッジ」
ケーブルに流れ得るデータの中継機能を持った装置。
『プロキシサーバー』
コンピュータに代わってインターネットに接続しサイトへアクセスするサーバー

『SMTP』
simple mail transfer protocol
『POP』
Post Office Protocol 名前などを伝えた上でメールを受け取る仕組み
『IMAP』
Internaet Message Access Protocol POPha受信すると消えるが、IMAPはキャッシュとしてアクセスする。

『セッション』
通信の開始から終了まで
『クッキー』
訪問ユーザーの情報を一時的に記憶する仕組み
『キャッシュ』
直近に読み込んだ物を複製しておくこと。
『チューニング』
負荷がある部分を効率化させること。
『スロークエリり』
DBにおいて、時間のかかっている遅いSQLのことをよぶ。
『インデックス』
データベースのアクセス効率を向上させるために列に対して設定される。
『フロー』
データの流れを表す。
『単体テスト』
プログラムの比較的小さな単位の個々の機能をテストする。
『結合テスト』
手続きや関数といった個々の機能を結合させてテストさせる。

『チケット管理ツール』
タスク管理ツール。
『タスク』
仕事、職務、任務などの意味

「Source Tree」

『コンフリクト』

『デグレ』

『マージツール』

『ロジック』

『デバック』

『開発手法』
アジャイル開発は、小単位での実装とテストを繰り返して開発に対し、ウォーターフォール開発は、全体から小規模、という流れ
『アルゴリズム』

『フローチャート』

『設計図』

『要件定義』

課題確認、同業務を変えるのか、どんなシステムが役立つのか、条件、導入システム定義
『単体テスト』
メソッドごとにテストをする軟体テスト、結合テストという、他のシステムと結合してテストをする、総合テストは全体がスペックに基づいているかをみるテスト
